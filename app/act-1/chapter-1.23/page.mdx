# Chapter 1.23: Decoders and Encoders

<ChapterCard>
### The Briefing: Minterm Generators
A **Decoder** is a combinational circuit that converts binary information from $n$ input lines to a maximum of $2^n$ unique output lines. 
If the $n$-bit input value is $k$, exactly the $k$-th output line becomes active (either going High in an Active-High decoder, or going Low in an Active-Low decoder).
Functionally, a decoder is a **minterm generator**. A 3-to-8 decoder produces all 8 minterms for $A,B,C$. By passing the desired outputs through an external OR gate, you can construct ANY Boolean function.

An **Encoder** performs the inverse operation: converting $2^n$ input lines into an $n$-bit binary code. A **Priority Encoder** handles cases where multiple inputs are active simultaneously by only encoding the input with the highest designated priority.
</ChapterCard>

<GateTrap>
**Trap: Active-Low Cascading**
GATE questions frequently ask you to build a 4-to-16 decoder using five 2-to-4 decoders. The trick lies in the Enable pins. If the decoders are Active-Low, you must ensure you invert the enable logic properly or use the outputs of the main 2-to-4 decoder directly into the active-low enable pins of the subsequent decoders. Do not assume positive logic if the chip has bubbles on its outputs/enables.
</GateTrap>

<ArchitectShortcut>
**Shortcut: Decoder to SOP Mapping**
If asked to implement $F(A,B,C) = \Sigma m(1, 3, 5, 7)$ using a 3-to-8 decoder, you simply wire outputs $1, 3, 5,$ and $7$ into a single OR gate. 
If the decoder is Active-Low (NAND gates), you wire those same outputs into a single **NAND** gate to get the exact same function $F$ (due to De Morgan's: $(m_1' m_3' m_5' m_7')' = m_1 + m_3 + m_5 + m_7$).
</ArchitectShortcut>
