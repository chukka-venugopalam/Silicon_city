# Chapter 1.3: IEEE 754 Architecture

<ChapterCard>
### The Briefing: The Floating Point Standard
To represent vast dynamic ranges of real numbers, architectures employ the IEEE 754 standard. A single-precision (32-bit) floating-point number is partitioned into three fields:
1.  **Sign Bit ($S$):** 1 bit. `0` for positive, `1` for negative.
2.  **Biased Exponent ($E$):** 8 bits. Uses an excess-127 bias. Actual exponent $e = E - 127$.
3.  **Mantissa/Fraction ($M$):** 23 bits. Represents the fractional part of a normalized number (an implicit leading `1.` is assumed).

The value is calculated as:
$$V = (-1)^S \times (1.M) \times 2^{E-127}$$
</ChapterCard>

<GateTrap>
**Trap: The Implicit '1' Assumption**
A classic mistake is applying the implicit `1.` to all cases. IEEE 754 has special values. When the biased exponent $E = 0$ (all zeros), the number is **Subnormal**. The implicit leading bit becomes `0.`, and the exponent is fixed at $-126$, NOT $-127$.
Value for $E=0$: $$V = (-1)^S \times (0.M) \times 2^{-126}$$
If $E = 255$ (all ones), it represents Infinity (if $M=0$) or NaN (if $M \neq 0$).
</GateTrap>

<ArchitectShortcut>
**Shortcut: Fast Exponent Bias Math**
When given a decimal number like $12.5$, immediately convert to binary: $1100.1$. Normalize it to $1.1001 \times 2^3$.
The actual exponent is $3$. To find the 8-bit biased exponent $E$, do NOT do $127 + 3 = 130$ and then convert $130$ to binary from scratch.
Instead, $127$ in binary is `01111111`. Adding $3$ (`00000011`) simply rolls the lower bits. 
Even faster: $127 = 128 - 1$. So $127 + 3 = 130$. $128$ is `10000000`, add $2$ (`00000010`) $\rightarrow$ `10000010`.
</ArchitectShortcut>
